Architectural Scale

Parameters vs Return Data

Parameters (Inputs)

Act as localized internal placeholder variables defined directly inside the definition parameters. They accept incoming data during execution.

Return Statements (Outputs)

Pass computed results directly out of the local scope boundary back to the tracking script framework so it can be assigned inside expressions.

Once a function triggers a return execution path, it halts operations instantly and exits.
